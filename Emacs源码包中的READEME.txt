This directory tree holds version 23.4 of GNU Emacs, the extensible,
customizable, self-documenting real-time display editor.
这个目录树包含GNU Emacs 23.4，Emacs是可扩展的，可定制的，自记录实时显示编辑器。

The file INSTALL in this directory says how to build and install GNU Emacs on various systems, once you have unpacked or checked out the entire Emacs file tree.
INSTALL文件介绍如何在不同的操作系统上解压缩和安装Emacs。

See the file etc/NEWS for information on new features and other user-visible changes in recent versions of Emacs.
etc/NEWS文件中包含Emacs最近版本中的新特性和其他用户可见的改变。

The file etc/PROBLEMS contains information on many common problems that occur in building, installing and running Emacs.
etc/PROBLEMS文件包含在编译、安装和运行Emacs过程中发生的常见问题的信息。

You may encounter bugs in this release.  If you do, please report them; your bug reports are valuable contributions to the FSF, since they allow us to notice and fix problems on machines we don't have, or in code we don't use often.  Please send bug reports to the mailing list bug-gnu-emacs@gnu.org.  If possible, use M-x report-emacs-bug.
你可能会在该发行版本中遇到bug，如果确实遇到，请报告它们；你的bug报告对于FSF来说是有价值的贡献。因为它们让我们发现并且修改那些在我们没有的机器上或者那些我们不经常使用的代码中所发生的bug。请发送bug报告到邮件列表bug-gnu-emacs@gnu.org，若可能的话，请使用M-x report-emacs-bug（译注：这是命令行）。

See the "Bugs" section of the Emacs manual for more information on how to report bugs.  (The file `BUGS' in this directory explains how you can find and read that section using the Info files that come with Emacs.)  See `etc/MAILINGLISTS' for more information on mailing lists relating to GNU packages.
在Emacs手册中的Bugs小节查找更多关于如何报告bugs的信息。（该目录下的BUGS文件说明你如何能够找到和阅读那个小节通过使用Emacs自带的info文件。）在etc/MAILINGLISTS中获得更多关于GNU 包的邮件列表。

The `etc' subdirectory contains several other files, named in capital letters, which you might consider looking at when installing GNU Emacs.
etc 子目录包含以几个大写字母命名的文件，，当安装GNU Emacs的时候你可能考虑看一下它们。

The file `configure' is a shell script to acclimate Emacs to the oddities of your processor and operating system.  It creates the file `Makefile' (a script for the `make' program), which automates the process of building and installing Emacs.  See INSTALL for more detailed information.
Configure文件是一个使Emacs适应你的处理器和操作系统的shell脚本。它产生Makefile文件，makefile文件时的编译和安装Emacs的过程自动化。查看INSTALL获得更多信息。

The file `configure.in' is the input used by the autoconf program to construct the `configure' script.  Since Emacs has some configuration requirements that autoconf can't meet directly, and for historical reasons, `configure.in' uses an unholy marriage of custom-baked configuration code and autoconf macros.  If you want to rebuild `configure' from `configure.in', you will need to install a recent version of autoconf and GNU m4.
configure.in文件是被autoconf程序使用。autoconf程序构造configure脚本。因为Emacs有一些配置需求autoconf不能直接满足，还出于历史原因，configure.in使用了一个由手动配置和autoconf宏组成的不合理的组合。

The file `Makefile.in' is a template used by `configure' to create `Makefile'.
Makefile.in是configure创建Makefile文件时使用的一个模板。

The file `make-dist' is a shell script to build a distribution tar file from the current Emacs tree, containing only those files appropriate for distribution.  If you make extensive changes to Emacs, this script will help you distribute your version to others.
make-dist文件是用来从当前Emacs树产生分发tar文件的脚本，只包含那些适合分发的文件。如果你对Emacs进行扩展，该脚本能够帮助你将你的版本分发到其他人。

There are several subdirectories:
下面是几个子目录：

`src':holds the C code for Emacs (the Emacs Lisp interpreter and its primitives, the redisplay code, and some basic editing functions).
`src':包含Emacs的C代码（Emacs Lisp 解释器和它的一些基本命令，重现代码和其他一些基本的编辑函数）

`lisp':holds the Emacs Lisp code for Emacs (most everything else).
`lisp':包含Emacs的lisp代码（几乎尽在其中）

`leim':holds the library of Emacs input methods, Lisp code and auxiliary data files required to type international characters which can't be directly produced by your keyboard.
`leim':包含Emacs输入方法库，lisp代码和输入那些不能有你的键盘产生的国际字符所需辅助数据文件。

`lib-src':holds the source code for some utility programs for use by or with Emacs, like movemail and etags.
`lib-src':包含一些由Emacs使用或与Emacs协同使用的公共程序（如movemail和etags）的源代码。

`etc':holds miscellaneous architecture-independent data files Emacs uses, like the tutorial text and tool bar images. The contents of the `lisp', `leim', `info', and `doc' subdirectories are architecture-independent too.
`etc':含有Emacs所使用的各种和体系结构无关的数据文件，如本教程的文本和图像工具栏。`lisp', `leim', `info', and `doc'子目录包含的内容都是和体系结构有关的。

`info':holds the Info documentation tree for Emacs.
`info':包含了Emacs的info文档树。

`doc/emacs':holds the source code for the Emacs Manual.  If you modify the manual sources, you will need the `makeinfo' program to produce an updated manual.  `makeinfo' is part of the GNU Texinfo package; you need a suitably recent version of Texinfo.

`doc/emacs':包含了Emacs手册的源码。如果你修改了手册源码，你需要makeinfo程序来产生一个已更新的手册。makeinfo是GNU Texinfo包的一部分，你需要一个适当的最近版本的Texinfo。
`doc/lispref':holds the source code for the Emacs Lisp reference manual.
`doc/lispref':包含Emacs Lisp reference manual的源码。

`doc/lispintro':holds the source code for the Introduction to Programming in Emacs Lisp manual.
`doc/lispintro':包含Introduction to Programming in Emacs Lisp manual的源码。

`msdos':holds configuration files for compiling Emacs under MSDOG.
`msdos':包含在MSDOS下编译Emacs所需要的配置文件。

`nextstep':holds instructions and some other files for compiling the Nextstep port of Emacs, for GNUstep and Mac OS X Cocoa.
`nextstep':包含Emacs Nextstep端口介绍和一些其他的文件。

`nt':holds various command files and documentation files that pertain to building and running Emacs on Windows 9X/ME/NT/2000/XP.
`nt':包含Windows 9X/ME/NT/2000/XP编译和运行Emacs的各种命令文件和文档文件

`test':holds tests for various aspects of Emacs's functionality.
`test':包含用于Emacs函数各个方面的测试。

Building Emacs on non-Posix platforms requires tools that aren't part
of the standard distribution of the OS.  The platform-specific README
files and installation instructions should list the required tools.
非POSIX平台编译Emacs需要的工具，不属于该OS的标准分发。特定于平台自述文件和安装说明应列出所需的工具。
